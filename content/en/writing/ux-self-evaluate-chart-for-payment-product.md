This is one of my self-evaluate tools when designing payment-related products.

</aside>

| Topic | Keywords |
| --- | --- |
| **New User Guide** | The new user can understand the writing on the page |
|  | The new user can see all available payment methods |
|  | The new user can select his payment method fast & easily |
| **Returned User Guide** | The returned user can use his used payment method from last time as a default |
|  | The returned user can use other payment methods |
| **Payment Methods** | The user can prepare his payment method before payment |
|  | The user can compare all available payment methods |
|  | The system can recommend a payment method for the user |
|  | When there is only one payment method, skip the selecting step |
|  | When a payment method is unavailable, the user can know why it's unavailable |
| **Cards** | The user can choose not to save a card |
|  | The user can use OCR to input card info |
|  | The system can identify BIN automatically while inputting digits |
|  | Only required the minimize must-have info from the user |
| **Payment Safety** | The product need to have a secured RC function (Payment passcode, Biotic pass, OTP, etc.) |
|  | A smart & tech feel during the payment, such as OCR |
|  | Prohibit screen shot, refresh the code if needed |
|  | When signing an Auto Debit agreement, the user need to be noticed & know what he is doing |
|  | Saving card needs confirmation |
|  | The PIN pad needs security reminder (That it's not a native number pad) |
|  | Extra security check |
| **UX Details** | PIN pad / number pad do not need the user to click confirm after inputting |
|  | Prepare some common used amount for the user if needed |
| **Display a Payment Method** | Name |
|  | Acceptance mark |
|  | Promotional info |
|  | Availability |
|  | Simple how to |
|  | A using expectation |
| **Display the Transaction Related Info** | Clear & simple order summary |
|  | Stand out Total to Pay amount |
|  | Stand out promotional info |
|  | Pay to info |
|  | Cash source is recommended to display |
|  | Order expiration time |
|  | Exchange rate |
| **Display the Promotional Info** | The promotional info need to be displayed throughout the payment flow |
|  | The promotional info needs to fit local habits |
|  | The promotional info is clear & understandable |
| **Payment Result** | The user can pay again after a failure |
|  | There should have some after-payment conversion possibility |
| **DA** | The PSR data for a normal shopping scenario |
|  | Willingness analysis |
|  | Analyze user behaviors by two groups: new & returned |
|  | Payment method PSR analysis |
|  | User amount by payment methods |
|  | Payment failure reason analysis |
